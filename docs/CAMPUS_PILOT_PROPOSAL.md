# CityBrain 학생식당 AI 혼잡도 파일럿 제안서

## 1. 제안 배경

CityBrain은 학생식당의 혼잡도, 잔여 수량, 예상 대기시간 등 학생들이 식사 전 알고 싶어 하는 정보를 제공하고, 학교와 운영사에는 운영 개선에 활용 가능한 데이터를 제공하기 위한 스마트캠퍼스 MVP입니다.

기존 학생식당 운영에서는 학생이 직접 식당에 가보기 전까지 현재 혼잡도나 대기 상황을 알기 어렵습니다. 또한 학교와 운영사 입장에서도 시간대별 혼잡도, 학생 반응, 소진 패턴을 데이터로 확인하기 어렵습니다.

CityBrain은 이 문제를 해결하기 위해 학생 안내 화면, 관리자 리포트, YOLO 기반 혼잡도 추정, 혼잡도 기록 저장, CSV 내보내기 기능을 단계적으로 구현했습니다.

## 2. 파일럿 목적

본 파일럿은 정식 서비스 도입이 아니라, 학생식당에서 AI 기반 혼잡도 안내가 실제로 학생과 운영자에게 도움이 되는지 검증하는 것을 목적으로 합니다.

검증 목표는 다음과 같습니다.

- 학생이 식당 방문 전 혼잡도를 확인할 수 있는가
- 운영자가 시간대별 혼잡도 변화를 확인할 수 있는가
- 혼잡도 데이터가 운영 판단과 학생 안내에 활용될 수 있는가
- 카메라 기반 혼잡도 추정이 키오스크/업체 연동 없이도 대안 데이터가 될 수 있는가

## 3. 파일럿 범위

권장 운영 기간은 1주일입니다.

예시 운영 시간:

    월요일 ~ 금요일
    11:30 ~ 13:30

운영 장소는 학생식당 입구 또는 대기열이 확인 가능한 위치를 기준으로 합니다. 실제 CCTV 연동 전에는 웹캠 또는 테스트용 카메라로 대체할 수 있습니다.

## 4. 시스템 구성

    Webcam / RTSP Camera
    -> YOLO Person Detection
    -> People Count / Congestion Level
    -> CityBrain Backend API
    -> Student Vision Status Page
    -> Admin Vision Report
    -> Vision History Log
    -> CSV Export

주요 화면과 API는 다음과 같습니다.

- YOLO Vision Module: http://127.0.0.1:8081/api/congestion/latest
- CityBrain Vision API: http://127.0.0.1:8080/api/vision/congestion
- Student Page: http://127.0.0.1:8080/student/vision-status
- Admin Report: http://127.0.0.1:8080/admin/vision-report
- History Page: http://127.0.0.1:8080/admin/vision-history
- CSV Export: http://127.0.0.1:8080/api/vision/history/export.csv

## 5. 현재 구현 상태

현재 CityBrain은 다음 기능을 포함합니다.

- YOLO 기반 사람 인식 데모
- CityBrain 백엔드와 YOLO 모듈 연동
- 학생용 AI 혼잡도 화면
- 관리자용 AI 혼잡도 운영 리포트
- 혼잡도 기록 저장
- 혼잡도 기록 CSV 내보내기
- YOLO 연결 실패 시 fallback 응답

## 6. 수집 데이터

| 항목 | 설명 |
|---|---|
| person_count | YOLO가 인식한 사람 수 |
| congestion | 여유 / 보통 / 혼잡 등 혼잡도 상태 |
| updated_at | YOLO 모듈이 갱신한 시각 |
| saved_at | CityBrain이 기록을 저장한 시각 |
| source | 카메라 또는 비전 모듈 출처 |
| method | RTSP/Webcam + YOLO person detection |

## 7. 수집하지 않는 데이터

본 파일럿은 다음 데이터를 수집하거나 저장하지 않는 방향을 원칙으로 합니다.

- 영상 원본
- 얼굴 이미지
- 개인 식별 정보
- 학생 개인별 이동 경로
- 개인 추적 데이터
- 음성 데이터

본 시스템은 개인 식별이 아니라, 사람 수 기반 혼잡도 통계값 산출을 목표로 합니다.

## 8. 기대 효과

### 학생 측면

- 식당 방문 전 혼잡도 확인 가능
- 대기 시간이 길 경우 방문 시간 조정 가능
- 불필요한 헛걸음 감소

### 학교 측면

- 학생 복지 개선 근거 확보
- 혼잡 시간대 파악
- 스마트캠퍼스 실증 사례 확보

### 운영사 측면

- 시간대별 혼잡도 확인
- 피크 시간 운영 대응
- 운영 리포트 기반 개선 가능

## 9. 파일럿 성공 기준

- 학생 화면에서 혼잡도 정보가 정상 표시된다.
- 관리자 리포트에서 현재 혼잡도와 사람 수를 확인할 수 있다.
- 혼잡도 기록이 저장된다.
- CSV로 기록을 내보낼 수 있다.
- 영상 원본 저장 없이 통계값 중심으로 운영 가능하다.

## 10. 결론

CityBrain은 단순한 학생식당 앱이 아니라, 학생식당의 운영 데이터를 수집하고 학생 안내와 운영 리포트로 연결하는 스마트캠퍼스 운영 데이터 MVP입니다.

본 파일럿은 정식 도입을 전제로 하지 않고, 1주일 동안 제한적으로 AI 혼잡도 안내와 운영 리포트의 실효성을 검증하는 것을 목표로 합니다.
