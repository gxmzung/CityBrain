# CityBrain 비전 기반 혼잡도 추정 개인정보·영상 처리 원칙 초안

## 1. 문서 목적

이 문서는 CityBrain의 YOLO 기반 혼잡도 추정 기능을 학교 파일럿으로 검토할 때 필요한 개인정보 및 영상 처리 원칙을 정리한 초안입니다.

본 문서는 법률 자문 문서가 아니며, 실제 운영 전에는 학교의 개인정보보호 담당 부서, 시설 관리 부서, 학생식당 운영 주체와 추가 검토가 필요합니다.

## 2. 기본 원칙

CityBrain 비전 모듈은 개인 식별이 아니라 혼잡도 통계값 산출을 목적으로 합니다.

핵심 원칙은 다음과 같습니다.

- 얼굴 인식 금지
- 개인 식별 금지
- 개인 추적 금지
- 영상 원본 저장 금지
- 사람 수 기반 통계값만 활용
- 파일럿 운영 범위 최소화
- 학교 운영 기준에 따른 안내문 부착 검토

## 3. 처리하는 데이터

| 항목 | 설명 |
|---|---|
| 실시간 카메라 프레임 | YOLO 사람 인식에 사용되는 일시적 입력 |
| person_count | 인식된 사람 수 |
| congestion | 여유 / 보통 / 혼잡 등 혼잡도 상태 |
| updated_at | 인식 결과 갱신 시각 |
| saved_at | 기록 저장 시각 |
| source | 카메라 또는 모듈 출처 |
| method | RTSP/Webcam + YOLO person detection |

## 4. 저장하지 않는 데이터

본 파일럿에서는 다음 데이터를 저장하지 않는 것을 원칙으로 합니다.

- 영상 원본
- 이미지 캡처
- 얼굴 이미지
- 개인별 식별값
- 학생 개인 정보
- 개인 이동 경로
- 음성 정보
- 얼굴 특징값 또는 생체정보

## 5. 영상 처리 방식

CityBrain의 비전 모듈은 카메라 입력을 받아 YOLO 객체 인식을 수행하고, 사람 수를 기반으로 혼잡도를 계산합니다.

    Camera Frame
    -> YOLO Person Detection
    -> People Count
    -> Congestion Level
    -> Statistics Only

처리 결과는 사람 수와 혼잡도 통계값으로만 변환됩니다.

## 6. 운영 안내문 예시

    CityBrain 혼잡도 안내 파일럿 운영 안내

    본 구역에서는 학생식당 혼잡도 안내를 위한 AI 기반 사람 수 통계 측정 파일럿이 진행될 수 있습니다.

    - 목적: 학생식당 혼잡도 안내 및 운영 개선 검토
    - 방식: 카메라 영상에서 사람 수 통계값만 산출
    - 저장 데이터: 사람 수, 혼잡도, 측정 시각
    - 저장하지 않는 데이터: 영상 원본, 얼굴 이미지, 개인 식별 정보
    - 운영 기간: 파일럿 기간 중 제한 운영
    - 문의: 담당 부서 또는 운영 담당자

## 7. 접근 권한

| 역할 | 접근 가능 항목 |
|---|---|
| 학생 | 현재 혼잡도, 예상 대기 안내 |
| 운영자 | 현재 혼잡도, 최근 기록, CSV 리포트 |
| 관리자 | 시스템 설정, 운영 기록, 장애 확인 |

MVP 단계에서는 로컬 환경에서 검증하며, 정식 운영 전에는 인증과 권한 관리를 추가해야 합니다.

## 8. 위험 요소

실제 학교 환경에서 운영하려면 다음 위험 요소를 검토해야 합니다.

- CCTV 접근 권한
- 영상정보처리기기 운영 기준
- 학생 고지 방식
- 영상 원본 저장 여부
- 외부 서버 전송 여부
- 카메라 위치와 촬영 범위
- 얼굴 또는 개인 식별 가능성
- 운영자 접근 권한 관리

## 9. 완화 방안

- 영상 원본 저장 비활성화
- 사람 수 통계값만 저장
- 얼굴 인식 기능 사용 금지
- 카메라 각도 최소화
- 안내문 부착
- 파일럿 기간 제한
- 운영자 접근 제한
- 정식 운영 전 학교 담당 부서 검토

## 10. 결론

CityBrain 비전 모듈은 개인 식별 기술이 아니라, 학생식당 혼잡도 안내를 위한 사람 수 기반 통계 산출 모듈입니다.

다만 학교 현장에서 카메라 또는 CCTV를 활용하는 경우에는 반드시 영상정보 처리 기준, 안내문, 접근 권한, 보관 정책을 사전에 검토해야 합니다.
