# 보안·백업·장애 대응 계획

## 현재 V4 반영 사항

- 관리자 계정 비밀번호 PBKDF2 해시 저장
- 관리자 쿠키 `httponly`, `samesite=lax`
- 감사 로그 테이블 저장
- 로그 CSV 다운로드
- SQLite 백업 스크립트 제공: `scripts/backup_sqlite.py`
- 운영 상태 API: `/api/ops/status`

## 정식 운영 전 보강 사항

- HTTPS 필수 적용
- 관리자 2FA 적용
- 학교 SSO 또는 OIDC 적용
- 운영자/조회자/관리자 권한 분리 고도화
- 주기적 DB 백업 자동화
- 장애 알림 채널 구성
- 로그 보존 정책 확정

## 장애 대응 시나리오

### 1. 서버 다운
1. `/api/health` 확인
2. 프로세스 재시작
3. 최근 배포 변경 내역 확인
4. DB 파일 존재 여부 확인
5. 복구 실패 시 최근 백업으로 롤백

### 2. 데이터 오입력
1. 관리자 로그 확인
2. 상태값 수동 복구
3. 필요 시 `reset-demo` 또는 백업 DB 복구
4. 운영자 교육/권한 조정

### 3. 개인정보 관련 이슈
1. 서비스 임시 중지
2. 접근 로그 보존
3. 학교 개인정보보호 담당자 보고
4. 원인 분석 후 재발 방지 조치
