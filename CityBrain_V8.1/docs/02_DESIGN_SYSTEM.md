# CityBrain 디자인 시스템 V4

## 목표

학교 공식 서비스처럼 보이되, 학생이 바로 이해할 수 있는 식사 앱 UX를 우선한다.

## 핵심 원칙

1. 첫 화면에서 3초 안에 `남은 수량 / 혼잡도 / 대기시간`이 보여야 한다.
2. 색상은 오렌지 식사 서비스 톤 + 배재대 블루 운영 플랫폼 톤으로 분리한다.
3. 학생 화면은 모바일 앱처럼 보이게 하고, 관리자 화면은 학교 운영 대시보드처럼 보이게 한다.
4. 카드 반경, 여백, 글자 크기 규칙을 고정해 화면이 덕지덕지 붙어 보이지 않게 한다.

## 컬러 토큰

| 토큰 | 값 | 용도 |
|---|---|---|
| Brand Blue | `#0A4DA2` | 관리자/학교 공식성 |
| Brand Blue Dark | `#083B7D` | 헤더/강조 |
| Accent Orange | `#F1A143` | 학생 식사 화면 메인 |
| Success Green | `#17A668` | 원활/이용 가능 |
| Warning Orange | `#F0B44D` | 보통/주의 |
| Danger Red | `#E05C5C` | 혼잡/위험 |
| App BG | `#F6F8FB` | 기본 배경 |
| Text Primary | `#171B24` | 본문 핵심 텍스트 |

## 컴포넌트

- Student Top Bar: 오렌지, 제목 `식사`, 실시간 안내 부제
- Date Selector: 날짜 전환 바
- Meal Tabs: 아침/점심/저녁
- Filter Chips: 거리/가격/영업 중/리뷰/평점
- Hero Food Card: 오늘의 일품식, 남은 수량, 소진 예상, 혼잡도, 대기시간
- Admin Metric Card: 총 수량, 잔여 수량, 판매량, 혼잡도
- Status Pill: 원활/보통/혼잡 상태 표현

## 납품 전 추가 QA

- 작은 Android 화면에서 카드 줄바꿈 확인
- 실제 폰 밝기 낮은 상태에서 대비 확인
- VoiceOver/TalkBack 읽기 순서 확인
- 혼잡도 색상만으로 의미를 전달하지 않도록 텍스트 병행
- 네트워크 오류 시 빈 화면 대신 오프라인 예비값 표시
