# Demo Flow Evidence

## Purpose

이 폴더는 CityBrain의 90초/3분 시연 흐름을 정리하는 공간이다.

---

## 90-Second Flow

```text
1. 학생 화면 접속
2. 오늘 메뉴 확인
3. 혼잡도와 남은 수량 확인
4. 만족도 또는 반응 입력
5. 관리자 화면 접속
6. 메뉴 또는 혼잡도 상태 입력
7. 학생 화면에 반영되는 흐름 설명
8. 학교 측 ROI 설명
```

---

## 3-Minute Flow

```text
1. 문제 제기
   - 학생은 식당에 직접 가보기 전까지 메뉴 상태, 혼잡도, 소진 여부를 알기 어렵다.

2. 학생 화면 시연
   - 오늘 메뉴 확인
   - 혼잡도 확인
   - 남은 수량 또는 소진 상태 확인
   - 만족도 입력

3. 관리자 화면 시연
   - 메뉴 등록 또는 수정
   - 혼잡도 입력
   - 소진 상태 입력
   - 학생 반응 확인

4. 데이터 흐름 설명
   - 학생은 조회한다.
   - 관리자는 입력한다.
   - FastAPI는 연결한다.
   - SQLite는 저장한다.
   - Android와 Web은 보여준다.

5. 학교 측 ROI 설명
   - 학생 편의가 아니라 운영 데이터가 핵심이다.
   - 메뉴 만족도, 혼잡 시간대, 소진 불만, 학생 반응이 누적된다.
   - 식수 예측, 메뉴 수량 조절, 혼잡 분산, 민원 감소에 활용 가능하다.

6. 한계점과 확장 방향
   - 초기 MVP는 운영자 입력과 학생 체감 데이터 기반이다.
   - 추후 키오스크 로그, 결제 건수, 시간대별 방문 패턴과 연동 가능하다.
```

---

## Final Sentence

```text
CityBrain의 학교 측 ROI는 이미지가 아니라 운영 데이터입니다.
```
