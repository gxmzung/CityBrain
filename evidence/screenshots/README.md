# Screenshots Evidence

## Purpose

이 폴더는 CityBrain의 실제 구현 화면 캡처를 정리하는 공간이다.

심사위원에게 보여줄 화면은 단순히 예쁜 UI가 아니라,  
학생 화면과 관리자 화면이 실제로 분리되어 있고, 서비스 흐름이 구현되어 있다는 증거가 되어야 한다.

---

## Required Screenshots

아래 화면을 우선 확보한다.

```text
1. 학생 홈 화면
2. 오늘 메뉴 화면
3. 혼잡도 또는 식당 상태 화면
4. 남은 수량 또는 소진 상태 화면
5. 만족도/반응 입력 화면
6. 관리자 로그인 화면
7. 관리자 메뉴 등록/수정 화면
8. 관리자 혼잡도 또는 소진 상태 입력 화면
9. 관리자 학생 반응 확인 화면
10. Android 앱 화면
```

## Presentation Use

발표에서는 모든 화면을 다 보여줄 필요는 없다.

우선순위는 다음과 같다.

```text
1. 학생 화면
2. 관리자 화면
3. 데이터가 반영되는 흐름
4. Android 앱 연결 화면
```

핵심은 “학생은 조회하고, 관리자는 입력하고, 데이터는 운영 개선 근거로 누적된다”는 흐름을 보여주는 것이다.
